# M3 搜索流程问题诊断结果

## 核心问题

淘宝搜索接口返回错误页面：
```
https://error.taobao.com/app/tbhome/common/error.html?from=https://www.taobao.com/search
```

这表示直接通过 `https://www.taobao.com/search?q=` 的 GET 请求被淘宝拦截，很可能是：

1. **淘宝反爬虫机制**：检测到非浏览器请求或机器访问
   - Playwright 虽然是真实浏览器，但可能某些头信息或 UA 导致被识别为爬虫
   - 淘宝可能检测会话的异常行为

2. **会话验证失败**：即使 session 已保存，搜索请求仍被拦截
   - 可能需要额外的验证码或 token
   - Cookie 可能过期或不完整

3. **CloudFlare/Akamai 防护**：淘宝使用的 CDN 可能有额外的反爬检测

## 解决方案

### 方案 1：从首页搜索框发起请求（推荐）
**原理**：通过 UI 交互而非直接 URL 跳转，更符合人类行为

优点：
- 避免被检测为爬虫
- 所有必需的 headers 和 referrer 都由浏览器自动设置
- 符合淘宝的使用流程

实施：
1. 导航到 taobao.com 首页
2. 定位搜索框
3. 填充关键词
4. 按回车或点击搜索按钮
5. 等待导航完成

### 方案 2：添加反爬虫对抗
**如果首页搜索也被拦截**：
- 增加 `User-Agent` 轮换
- 添加随机延迟
- 使用代理 IP（如果有）
- 添加必要的请求头（`Referer`, `Accept-Language` 等）

### 方案 3：放宽测试要求
**临时方案**（如果反爬虫无法绕过）：
- 改用本地模拟数据测试后续流程
- 在真实淘宝页面上手动测试搜索
- 考虑使用淘宝官方 API（如果可用）

## 建议后续步骤

1. **立即实施**：改为首页搜索框交互（方案 1）
   - 这是最可靠的绕过反爬虫的方式
   - 改动最小

2. **测试**：本地跑一遍，看是否能通过反爬虫检测

3. **如果仍失败**：添加人类行为模拟（随机延迟、鼠标移动等）
