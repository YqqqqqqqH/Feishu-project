# 项目模块拆分

## 整体数据流

```
用户（飞书消息）
    ↓
[M1] 飞书接入层  ── 解析搜索词、筛选条件
    ↓
[M2] Agent 调度层  ── 任务编排、状态管理
    ↓
[M3] 浏览器自动化层  ── 登录、搜索、筛选、加购
    ↓
[M4] 数据处理层  ── 结构化商品信息、生成报告
    ↓
[M1] 飞书接入层  ── 回传卡片报告
```

---

## M1：飞书接入层

**职责**：作为系统的唯一入口和出口。

核心功能：
- 监听飞书消息（通过 Webhook 或 Event API 订阅）
- 从消息中提取任务参数（搜索词、筛选条件）
- 用 Bot Token 向指定会话回传报告

关键问题：
- Bot Token 有 2 小时时效，需要自动刷新
- 消息解析：支持关键词触发（如"搜索 索尼耳机"）或语义理解（接入 LLM 解析自然语言）
- 回传格式：飞书卡片消息（interactive），可展示商品名、价格、好评率、链接

待决策：
- [ ] 用关键词触发还是 LLM 语义解析？（后者更灵活，前者更稳定）
- [ ] 飞书 Bot 部署在哪里？需要一个公网可访问的 Webhook 地址

## M2：Agent 调度层

**职责**：连接飞书和浏览器，管理整个任务生命周期。

核心功能：
- 接收 M1 解析出的任务参数，构造执行上下文
- 按顺序调用 M3 的各个步骤
- 管理"等待用户输入"的状态（登录时需要暂停等用户扫码/输入验证码）
- 捕获异常，决定重试还是报错回传

关键问题：
- 这一层对应 openclaw 的 Skill 入口（skill.py）
- 需要定义清晰的输入参数（input schema）和输出结构（output schema）

待决策：
- [ ] 登录等待如何通知用户？（飞书发消息"请在 30 秒内完成登录"）
- [ ] 任务超时如何处理？

## M3：浏览器自动化层

**职责**：操控浏览器完成淘宝上的所有操作。

核心功能（按执行顺序）：
1. **启动浏览器**：Playwright 启动 Chromium，优先加载已保存的 session（cookies）
2. **登录**：
   - 若 session 有效，直接跳过登录
   - 若 session 失效，打开登录页，等待用户手动扫码/输入，完成后保存 session
3. **搜索**：在搜索框输入关键词，等待商品列表渲染完成（`wait_for_selector`）
4. **筛选**：遍历商品列表，获取好评率（可能需要进入详情页），筛选 ≥ 阈值的商品
5. **加购**：对符合条件的商品点击"加入购物车"，处理可能的弹窗

关键技术难点：
- **反爬/验证码**：淘宝有滑块验证，session 复用可以大幅减少触发频率
- **好评率位置不固定**：列表页可能不显示，需要进入详情页抓取，性能开销大
- **动态加载**：商品列表滚动加载，需要控制滚动并等待新内容渲染
- **元素定位脆弱**：淘宝 DOM 结构可能随时变化，选择器需要有备选方案

待决策：
- [ ] 好评率阈值是否可配置？（brainstorm 中提到可放宽，至少返回 3 个商品）
- [ ] 是否需要截图作为证据附在报告中？

## M4：数据处理层

**职责**：将浏览器抓取的原始数据转化为结构化报告。

核心功能：
- 商品信息结构化：名称、价格、好评率、销量、链接、店铺名
- 报告生成：根据商品类型自适应报告格式（brainstorm 中提到不固定模板）
- 可选：接入 LLM 对商品评价做摘要，生成推荐理由

输出结构示例：
```json
{
  "keyword": "索尼耳机",
  "threshold": 99,
  "total_found": 12,
  "qualified": 3,
  "items": [
    {
      "name": "索尼 WH-1000XM5 头戴式降噪耳机",
      "price": 2299,
      "rating": 99.2,
      "sales": 3400,
      "url": "https://...",
      "shop": "索尼官方旗舰店"
    }
  ]
}
```

待决策：
- [ ] 是否用 LLM 生成推荐理由？（加分项，但增加延迟）

## M5：Skill 定义层（openclaw 规范）

**职责**：将上述模块封装为符合 openclaw 规范的 Skill。

核心内容：
- `SKILL.md`：Skill 的元数据、输入/输出 schema、触发条件
- `skill.py`：Skill 入口，调用 M2~M4

输入参数（input schema）：
```yaml
keyword: string        # 搜索关键词（从飞书消息提取）
rating_threshold: int  # 好评率阈值，默认 99，可放宽至 95
max_results: int       # 最多返回商品数，默认 5
feishu_chat_id: string # 回传结果的飞书会话 ID
```

输出参数（output schema）：
```yaml
status: success | partial | failed
report: object   # M4 生成的结构化报告
message: string  # 人类可读的摘要
```

---

## 模块依赖关系

```
M5 (Skill 定义)
 └── M2 (调度层)
      ├── M1 (飞书接入)
      ├── M3 (浏览器自动化)
      └── M4 (数据处理)
```

M1 被 M2 调用两次：任务开始时接收参数，任务结束时回传报告。
